# Decision Agenda — Shared Pattern

> 모든 factory 스킬의 Step 0에서 참조하는 공유 의사결정 프로토콜.

## 의사결정 3-Tier 분류

| Tier | 이름 | 누가 결정 | 기준 |
|------|------|----------|------|
| **Tier 1** | Strategic | **사용자 필수** | 방향/범위/우선순위/접근법 선택, 하류 스킬에 연쇄 영향 |
| **Tier 2** | Tactical | AI 기본값 + 사용자 오버라이드 | 합리적 기본값 존재, 잘못 골라도 재작업 적음 |
| **Tier 3** | Execution | AI (투명 실행) | 결정론적, 의미 있는 대안 없음, 즉시 되돌릴 수 있음 |

## Step 0: Decision Agenda 프로토콜

### 0-1. Read Prior Decisions
이전 스킬의 decisions 파일을 읽기 (있으면):
- `factory/research/decisions.md`
- `factory/prd/decisions.md`
- `factory/analysis/decisions.md`
- `factory/architect/decisions.md`
- `factory/ui-design/decisions.md`
- `factory/tdd/decisions.md`
- `factory/develop/decisions.md`
- `factory/e2e/decisions.md`
- `factory/code-review/decisions.md`

이미 결정된 항목은 "Pre-decided"로 표시하고 skip.

### 0-2. Generate Agenda
입력과 컨텍스트를 분석하여 이 스킬에서 필요한 결정 목록 생성.
각 결정을 Tier 1 / Tier 2로 분류.

### 0-3. Agenda Overview
`AskUserQuestion`으로 진행 방식 확인:

| 선택지 | 설명 |
|--------|------|
| **핵심만 검토 (Recommended)** | Tier 1만 직접, Tier 2는 AI 기본값 |
| **전체 검토** | Tier 1 + Tier 2 모든 결정을 직접 |
| **알아서 해줘** | AI가 결정하고 근거 기록. 비위임(Non-delegatable) 항목만 확인 |

### 0-4. Decision Walk-through
- **Tier 1**: `AskUserQuestion` 배치 (최대 4개/배치, 의존성 순서 준수)
- **Tier 2**: "전체 검토" 선택 시에만 질문, 아니면 AI 기본값 적용
- **비위임 항목**: "알아서 해줘"에서도 반드시 사용자 확인

### 0-5. Record Decisions
`factory/<skill>/decisions.md`에 기록:

```markdown
# Decisions: [Skill Name]

> Generated by factory-<skill>

| ID | Decision | Choice | Rationale | Decided By | Date |
|----|----------|--------|-----------|------------|------|
| XX-D-001 | ... | ... | ... | User / AI | YYYY-MM-DD |
```

### 0-6. Proceed
모든 결정 기록 후 스킬의 실행 단계(기존 Step 1~)로 진행.

## "알아서 해줘" 프로토콜

Full delegation 시에도 각 스킬별 **비위임(Non-delegatable) 항목**은 반드시 사용자 확인:

| Skill | 비위임 항목 |
|-------|-----------|
| research | 리서치 토픽 선정 |
| prd | 성공 기준, 우선순위 트레이드오프 |
| analysis | 구현 접근법 선택 (A/B/C) |
| architect | 기술 스택, GO/NO-GO |
| ui-design | 디자인 방향, 화면 목록 승인 |
| tdd | TDD 예외 승인 |
| develop | 에스컬레이션 응답 |
| e2e | 커버리지 기준 |
| code-review | 판정 기준 오버라이드 |

AI가 전체 결정 후 요약 테이블을 보여주고, 사용자가 "좋습니다, 진행" 또는 "일부 수정"을 선택하는 확인 게이트를 거친다.

## 하류 스킬 규칙

- 실행 전 상류 스킬의 `decisions.md`를 읽고 이미 결정된 항목 반영
- 재실행 시 해당 스킬의 `decisions.md`를 덮어쓰기
